מה בנינו — הושלם
🖥️
ממשק משתמש — Web App
הושלם
Next.js + TypeScript על Netlify. טופס העלאה עם drag-and-drop, תצוגה מקדימה של PDF, הגדרות הדפסה (עותקים, צבע, דו-צדדי), פרטי משתמש.
  • קובץ: apps/web/src/app/page.tsx
  • קובץ: apps/web/src/app/upload-form.tsx
🔌
API — יצירת עבודת הדפסה
הושלם
Route מלא: ולידציה של הטופס, העלאת PDF ל-Supabase Storage, יצירת שורה ב-print_jobs, token מאובטח לסטטוס.
  • קובץ: apps/web/src/app/api/jobs/create/route.ts
🗂️
סכמת Supabase
הושלם (צריך ניקוי)
טבלאות: print_jobs, print_agents, app_settings. Bucket: print-files. RLS וגישות הוגדרו.
  • תיקייה: supabase/migrations/
🛡️
פאנל אדמין
הושלם
כניסה מאובטחת, תור עבודות הדפסה, צפייה בסוכנים, הגדרות מערכת (אישור ידני, מגבלות גודל, passcode).
  • תיקייה: apps/web/src/app/admin/
📄
דף סטטוס לעבודת הדפסה
הושלם
אחרי שליחת טופס, המשתמש מנותב לדף /status/[jobId]?token=... שמציג את מצב ההדפסה בזמן אמת.
  • תיקייה: apps/web/src/app/status/
📬
מערכת ישנה — Email-to-Print
MVP מלא
שרת Node.js מקומי שסורק IMAP, מסנן לפי [PRINT] בנושא, מדפיס PDF/תמונות ישירות ל-Windows printer.
  • קבצים: server.js, index.html
  • פועל בנפרד מ-PrintDesk החדש
מה חסר — נדרש להשלמה
🤖
Print Agent — לוגיקת הדפסה
קריטי — חסר לחלוטין
הקובץ apps/agent/src/index.ts כרגע מדפיס רק "Phase 4 will add polling, claiming, downloading and printing." — הלוגיקה האמיתית לא קיימת.
  • ❌ סקירת print_jobs בסוג approved
  • ❌ Claim atomically לעבודה (מניעת כפילויות)
  • ❌ הורדת PDF מ-Supabase Storage
  • ❌ הדפסה ב-SumatraPDF (שקטה, ב-Windows)
  • ❌ עדכון סטטוס: printing → printed / failed
  • ❌ Heartbeat — Agent online/offline
🔑
ENV ב-Netlify — חסרים
קריטי — המערכת לא פועלת
הטופס שולח ל-/api/jobs/create אבל אם משתני הסביבה לא מוגדרים ב-Netlify — ה-API יחזיר שגיאה.
  • ❌ NEXT_PUBLIC_SUPABASE_URL
  • ❌ NEXT_PUBLIC_SUPABASE_ANON_KEY
  • ❌ SUPABASE_SERVICE_ROLE_KEY
  • ❌ ADMIN_EMAILS

יש להגדיר ב-Netlify → Site settings → Environment variables.
🗃️
ניקוי Migrations
חשוב
קיימות 6 קבצי migration כולם עם prefix 0001_ — זה יוצר בלבול ושגיאות ב-supabase db push.
  • ❌ יש לאחד לקובץ אחד: 0001_printdesk_schema.sql
  • ❌ לוודא שה-schema פרוס על Supabase Production
📦
התקנה ואריזה של ה-Agent
חסר
אין עדיין דרך נוחה להתקין את ה-agent על מחשב Windows במשרד.
  • ❌ .env עם SUPABASE_URL, AGENT_ID וכו'
  • ❌ סקריפט install.bat / setup.ps1
  • ❌ הרצה כ-Windows Service (pm2 / NSSM)
  • ❌ תיעוד התקנה
👁️
תצוגה מקדימה של PDF
חלקי
ה-upload-form.tsx כולל iframe לתצוגה מקדימה, אבל ב-Netlify הצפייה לא עובדת — ייתכן בגלל CSP או שגיאת הרשאות לקובץ blob.
  • ⚠️ לבדוק ולתקן את מדיניות ה-CSP
🔔
עדכון סטטוס בזמן אמת
חלקי
דף הסטטוס קיים, אבל לא ברור אם יש polling/realtime subscription ל-Supabase שמעדכן אוטומטית.
  • ⚠️ לוודא שיש Supabase Realtime subscription
  • ⚠️ או polling כל 5 שניות כ-fallback

ציר הזמן של הפיתוח
Phase 1 — הושלם ✓
MVP של Email-to-Print
שרת Node.js מקומי + ממשק גלאסמורפיזם. מסנן מיילים ב-IMAP ומדפיס קבצים ישירות.
Phase 2 — הושלם ✓
PrintDesk Web App
Next.js + Supabase. טופס העלאה, API ליצירת עבודה, פאנל אדמין, הגדרות מערכת. פרוס ב-Netlify.
Phase 3 — הושלם ✓
Infrastructure ו-Schema
סכמת Supabase מלאה, טיפוסי TypeScript, Admin auth, Storage bucket, RLS policies.
Phase 4 — חסר ❌
Print Agent — הלוגיקה האמיתית
Polling, Claiming, Download, Print, Status update, Heartbeat. זה הלב החסר של המערכת כולה.
Phase 5 — עתיד
Production Deployment
Agent מותקן כ-Windows Service, ENV ב-Netlify, בדיקת E2E, הדרכת משתמשים.
